Copy EC2 Cross-Account

Hướng dẫn copy EC2 instance từ account này sang account khác trong cùng AWS Organization bằng phương pháp AMI Share + KMS CMK re-encryption. Áp dụng khi EBS volume mã hóa bằng AWS Managed Key.

Phiên bản: v1.0 Test lần cuối: 2026-06-26 Owner: trungcln Trạng thái: ACTIVE Account: jp:prod → jp:uat Severity: P4

Mục tiêu & Trigger

Runbook này giải quyết vấn đề gì và khi nào phải dùng.

Mục tiêu

Copy toàn bộ EC2 instance (bao gồm OS disk + data disk) từ jp:prod sang jp:uat mà không làm gián đoạn production, sử dụng AMI Share kết hợp KMS CMK để vượt qua giới hạn AWS Managed Key.

⚠️
AWS Managed Key (aws/ebs) không thể share cross-account.

Nếu EBS volume mã hóa bằng AWS Managed Key, lệnh modify-image-attribute sẽ báo lỗi "Snapshots encrypted with the AWS Managed CMK can't be shared". Phải dùng Customer Managed Key (CMK) làm trung gian.

Điều kiện tiên quyết

Kiểm tra đầy đủ trước khi bắt đầu.

⚠️
Thao tác jp:prod — xác nhận 2 lần trước mỗi lệnh write.

Dù production instance không bị tắt, các lệnh tạo AMI, sửa KMS key policy, share AMI đều là write operations trên jp:prod.

→Quyền truy cập

  • SSO login jp:prod: aws sso login --profile jp:prod
  • SSO login jp:uat: aws sso login --profile jp:uat
  • Verify jp:prod: aws sts get-caller-identity --profile jp:prod → Account 347486023795
  • Verify jp:uat: aws sts get-caller-identity --profile jp:uat → Account 799374460549
  • Quyền cần thiết jp:prod: ec2:CopyImage, ec2:ModifyImageAttribute, ec2:ModifySnapshotAttribute, kms:PutKeyPolicy, kms:GetKeyPolicy
  • Quyền cần thiết jp:uat: ec2:CopyImage, ec2:RunInstances, ec2:CreateSecurityGroup, ec2:RevokeSecurityGroupEgress (AdministratorAccess đủ)

Các bước thực hiện

Thực hiện đúng thứ tự. Có 3 cách: Console · CLI · Terraform.

🔴
⚠️ Bước 1–5 chạy trên jp:prod — xác nhận 2 lần trước mỗi lệnh write.

Production instance KHÔNG bị tắt. Tuy nhiên, các thao tác KMS key policy và AMI share vẫn là write operations.

  1. 🟢 Chọn AMI nguồn (hoặc tạo mới nếu cần)

    Kiểm tra AMI sẵn có. Nếu AMI cũ quá (>2 tuần) hoặc cần snapshot tại thời điểm hiện tại, tạo AMI mới với --no-reboot (không tắt instance).

    🟢 readjp:prod
    # Xem AMI hiện có
aws ec2 describe-images \
  --profile jp:prod --region ap-southeast-1 \
  --owners self --filters "Name=name,Values=*webapp*" \
  --query 'Images[*].{ImageId:ImageId,Name:Name,CreationDate:CreationDate,State:State}' \
  --output table

# (Tuỳ chọn) Tạo AMI mới nếu cần snapshot tươi — ~20-40 phút
# 🔴 PROD write — xác nhận trước khi chạy
aws ec2 create-image \
  --instance-id i-0cb8bae3be071dc0e \
  --name "webapp-manual-backup-$(date +%Y%m%d)" \
  --no-reboot \
  --profile jp:prod --region ap-southeast-1
    AMI sẵn có State=available, hoặc create-image trả về ImageId mới
  2. 🔴 Copy AMI trong jp:prod — re-encrypt bằng CMK

    AWS Managed Key không share được cross-account. Phải copy AMI sang bản mới dùng Customer Managed Key (CMK) của prod. Thao tác này tạo AMI mới trong jp:prod, không ảnh hưởng instance.

    🔴 prod write · ~30-40 phútjp:prod
    aws ec2 copy-image \
  --source-region ap-southeast-1 \
  --source-image-id ami-042a2d6d54909e16c \
  --name "webapp-cmk-encrypted-for-uat-$(date +%Y%m%d)" \
  --description "Re-encrypted with CMK for UAT cross-account copy" \
  --encrypted \
  --kms-key-id bdd23e25-24a0-4fb0-954c-1cf087323490 \
  --profile jp:prod --region ap-southeast-1
# Lưu ImageId trả về → dùng ở bước 4-5

# Poll trạng thái (chờ available)
aws ec2 describe-images \
  --image-ids ami-<NEW-ID> \
  --profile jp:prod --region ap-southeast-1 \
  --query 'Images[0].State' --output text
    Trả về {"ImageId": "ami-xxx"} → poll đến khi State = "available"
  3. 🔴 Cập nhật KMS key policy — thêm UAT account

    Cho phép UAT account sử dụng CMK của prod để decrypt snapshot khi copy sang UAT. Lấy policy hiện tại rồi merge thêm statement mới.

    🔴 prod write · KMS key policyjp:prod
    # Xem policy hiện tại
aws kms get-key-policy \
  --key-id bdd23e25-24a0-4fb0-954c-1cf087323490 \
  --policy-name default \
  --profile jp:prod --region ap-southeast-1 --output text

# Tạo file policy mới (giữ statement cũ, thêm UAT)
cat > /tmp/cmk-policy-uat.json <<'EOF'
{
  "Version": "2012-10-17",
  "Id": "key-default-1",
  "Statement": [
    {
      "Sid": "Enable IAM User Permissions",
      "Effect": "Allow",
      "Principal": {"AWS": "arn:aws:iam::347486023795:root"},
      "Action": "kms:*",
      "Resource": "*"
    },
    {
      "Sid": "Allow UAT account to use CMK for AMI copy",
      "Effect": "Allow",
      "Principal": {"AWS": "arn:aws:iam::799374460549:root"},
      "Action": ["kms:DescribeKey","kms:ReEncrypt*","kms:GenerateDataKey*","kms:CreateGrant","kms:Decrypt"],
      "Resource": "*"
    }
  ]
}
EOF

# Apply policy
aws kms put-key-policy \
  --key-id bdd23e25-24a0-4fb0-954c-1cf087323490 \
  --policy-name default \
  --policy file:///tmp/cmk-policy-uat.json \
  --profile jp:prod --region ap-southeast-1
    Không có output = thành công. Verify: get-key-policy lại → phải thấy statement UAT
  4. 🔴 Share snapshots của AMI CMK với UAT

    Ngoài share AMI, cần share từng snapshot để UAT account có thể đọc dữ liệu khi copy.

    🔴 prod write · snapshot permissionjp:prod
    # Lấy danh sách snapshot của AMI CMK
aws ec2 describe-images \
  --image-ids ami-03ecb3d14486817c0 \
  --profile jp:prod --region ap-southeast-1 \
  --query 'Images[0].BlockDeviceMappings[?Ebs.SnapshotId!=null].Ebs.SnapshotId' \
  --output json

# Share từng snapshot (thay snap-xxx bằng ID thực tế)
for snap in snap-08ce815b09689f1fc snap-0d874052a3620cc9f; do
  aws ec2 modify-snapshot-attribute \
    --snapshot-id $snap \
    --attribute createVolumePermission \
    --operation-type add \
    --user-ids 799374460549 \
    --profile jp:prod --region ap-southeast-1
  echo "Shared $snap"
done
    Mỗi lệnh không có output = thành công
  5. 🔴 Share AMI CMK với UAT account

    Thêm UAT account vào LaunchPermission của AMI CMK.

    🔴 prod write · AMI permissionjp:prod
    aws ec2 modify-image-attribute \
  --image-id ami-03ecb3d14486817c0 \
  --launch-permission "Add=[{UserId=799374460549}]" \
  --profile jp:prod --region ap-southeast-1

# Verify
aws ec2 describe-image-attribute \
  --image-id ami-03ecb3d14486817c0 \
  --attribute launchPermission \
  --profile jp:prod --region ap-southeast-1
    LaunchPermissions = [{UserId: "799374460549"}]
  6. 🟡 Copy AMI vào UAT — re-encrypt bằng aws/ebs của UAT

    UAT tự copy AMI về account của mình, re-encrypt bằng AWS managed key của UAT. Từ bước này UAT hoàn toàn độc lập với key của prod.

    🟡 uat write · ~30-40 phútjp:uat
    aws ec2 copy-image \
  --source-region ap-southeast-1 \
  --source-image-id ami-03ecb3d14486817c0 \
  --name "jp-uat-ec2-webapp-from-prod-$(date +%Y%m%d)" \
  --description "Webapp copied from prod for UAT" \
  --encrypted \
  --profile jp:uat --region ap-southeast-1
# Lưu ImageId → dùng bước 8

# Poll trạng thái
aws ec2 describe-images \
  --image-ids ami-<UAT-ID> \
  --profile jp:uat --region ap-southeast-1 \
  --query 'Images[0].State' --output text
    State = "available" sau 20-40 phút
  7. 🟡 Tạo Security Group trong UAT

    Tạo SG trống, xóa outbound default. Cấu hình inbound/outbound theo nhu cầu sau khi instance lên.

    🟡 uat writejp:uat
    # Tạo SG
SG_ID=$(aws ec2 create-security-group \
  --group-name "jp-uat-sg-webapp" \
  --description "Webapp UAT - inbound/outbound managed manually" \
  --vpc-id vpc-05f30c1c1430f39a6 \
  --profile jp:uat --region ap-southeast-1 \
  --query 'GroupId' --output text)
echo "SG: $SG_ID"

# Xóa outbound default (all traffic)
aws ec2 revoke-security-group-egress \
  --group-id $SG_ID \
  --ip-permissions '[{"IpProtocol":"-1","IpRanges":[{"CidrIp":"0.0.0.0/0"}]}]' \
  --profile jp:uat --region ap-southeast-1
    SG ID trả về (sg-xxx), không có lỗi khi revoke
  8. 🟡 Launch EC2 instance trong UAT

    Launch instance từ AMI UAT đã copy. Dùng subnet private. IMDSv2 bắt buộc theo chuẩn bảo mật.

    🟡 uat write · launch instancejp:uat
    aws ec2 run-instances \
  --image-id ami-0ed77c0c5091e67b4 \
  --instance-type t3.medium \
  --subnet-id subnet-0f3f61756bac3e53d \
  --security-group-ids sg-030f2676223d7052e \
  --no-associate-public-ip-address \
  --tag-specifications \
    'ResourceType=instance,Tags=[{Key=Name,Value=jp-uat-ec2-webapp},{Key=Environment,Value=uat},{Key=SourceInstance,Value=i-0cb8bae3be071dc0e}]' \
  --metadata-options HttpTokens=required,HttpEndpoint=enabled \
  --profile jp:uat --region ap-southeast-1 \
  --query 'Instances[0].{InstanceId:InstanceId,PrivateIp:PrivateIpAddress,State:State.Name}'

# Chờ running
aws ec2 wait instance-running \
  --instance-ids i-<UAT-INSTANCE-ID> \
  --profile jp:uat --region ap-southeast-1
    State = "running", PrivateIp trong dải 10.0.1.0/24
  1. Đăng nhập Console và chọn account jp:prod

    Truy cập SSO portal → chọn account jp:prod (347486023795) → Management Console → region ap-southeast-1.

  2. Copy AMI nguồn với CMK (EC2 → AMIs)

    EC2 → Images → AMIs → chọn ami-042a2d6d54909e16c → Actions → Copy AMI.

    • Destination region: ap-southeast-1
    • Encryption: Enable
    • KMS key: chọn bdd23e25-24a0-4fb0-954c-1cf087323490 (EBS CMK)
    • Name: webapp-cmk-encrypted-for-uat-YYYYMMDD

    Bấm Copy AMI → đợi Status = available (~30-40 phút).

  3. Cập nhật KMS key policy (KMS → Customer managed keys)

    KMS → Customer managed keys → chọn key bdd23e25... → tab Key policy → Edit.

    Thêm statement sau vào mảng Statement[]:

    policy json — dán vào editor KMSjp:prod
    {
  "Sid": "Allow UAT account to use CMK for AMI copy",
  "Effect": "Allow",
  "Principal": {"AWS": "arn:aws:iam::799374460549:root"},
  "Action": ["kms:DescribeKey","kms:ReEncrypt*",
             "kms:GenerateDataKey*","kms:CreateGrant","kms:Decrypt"],
  "Resource": "*"

    Bấm Save changes.

  4. Share snapshots và AMI với UAT (EC2 → Snapshots / AMIs)

    Snapshot: EC2 → Elastic Block Store → Snapshots → lọc theo AMI ID vừa tạo → chọn từng snapshot → Actions → Modify permissions → Add account 799374460549.

    AMI: EC2 → AMIs → chọn AMI CMK → Actions → Edit AMI permissions → Add account ID 799374460549 → Save.

  5. Copy AMI vào UAT và launch instance

    Đổi sang account jp:uat (799374460549) → EC2 → AMIs → Private images → tìm AMI vừa được share.

    Chọn AMI → Actions → Copy AMI → Encryption: Enable (dùng aws/ebs mặc định) → đợi available.

    Sau khi available: chọn AMI → Launch instance from AMI.

    • Instance type: t3.medium
    • Network: VPC vpc-05f30c1c1430f39a6, subnet subnet-0f3f61756bac3e53d (private-a)
    • Security group: tạo mới hoặc chọn sg-030f2676223d7052e
    • Public IP: Disable
    • Metadata (Advanced): IMDSv2 Required

Terraform phù hợp cho lần đầu hoặc khi cần reproducible.

main.tf · ec2-cross-account-copyterraform
### providers.tf ###
provider "aws" {
  alias   = "prod"
  profile = "jp:prod"
  region  = "ap-southeast-1"
}
provider "aws" {
  alias   = "uat"
  profile = "jp:uat"
  region  = "ap-southeast-1"
}

### Bước 1: Data source — AMI nguồn trong prod ###
data "aws_ami" "webapp_source" {
  provider    = aws.prod
  most_recent = true
  owners      = ["self"]
  filter {
    name   = "name"
    values = ["*webapp*"]
  }
  filter {
    name   = "state"
    values = ["available"]
  }
}

### Bước 2: Copy AMI trong prod với CMK ###
resource "aws_ami_copy" "webapp_cmk" {
  provider          = aws.prod
  name              = "webapp-cmk-encrypted-for-uat-${formatdate("YYYYMMDD", timestamp())}"
  description       = "Re-encrypted with CMK for UAT cross-account copy"
  source_ami_id     = data.aws_ami.webapp_source.id
  source_ami_region = "ap-southeast-1"
  encrypted         = true
  kms_key_id        = "bdd23e25-24a0-4fb0-954c-1cf087323490"

  tags = {
    ManagedBy   = "terraform"
    Environment = "intermediate"
    Purpose     = "uat-migration"
  }
}

### Bước 3: KMS key policy — thêm UAT (dùng aws_kms_key_policy) ###
data "aws_iam_policy_document" "cmk_with_uat" {
  provider = aws.prod
  statement {
    sid       = "Enable IAM User Permissions"
    effect    = "Allow"
    principals { type = "AWS"; identifiers = ["arn:aws:iam::347486023795:root"] }
    actions   = ["kms:*"]
    resources = ["*"]
  }
  statement {
    sid    = "Allow UAT account to use CMK for AMI copy"
    effect = "Allow"
    principals { type = "AWS"; identifiers = ["arn:aws:iam::799374460549:root"] }
    actions   = ["kms:DescribeKey","kms:ReEncrypt*","kms:GenerateDataKey*","kms:CreateGrant","kms:Decrypt"]
    resources = ["*"]
  }
}

resource "aws_kms_key_policy" "cmk_uat_access" {
  provider = aws.prod
  key_id   = "bdd23e25-24a0-4fb0-954c-1cf087323490"
  policy   = data.aws_iam_policy_document.cmk_with_uat.json
}

### Bước 4-5: Share snapshot + AMI với UAT ###
resource "aws_ami_launch_permission" "share_to_uat" {
  provider   = aws.prod
  image_id   = aws_ami_copy.webapp_cmk.id
  account_id = "799374460549"
}

### Bước 6: Copy AMI vào UAT ###
resource "aws_ami_copy" "webapp_uat" {
  provider          = aws.uat
  name              = "jp-uat-ec2-webapp-from-prod-${formatdate("YYYYMMDD", timestamp())}"
  description       = "Webapp copied from prod for UAT"
  source_ami_id     = aws_ami_copy.webapp_cmk.id
  source_ami_region = "ap-southeast-1"
  encrypted         = true

  depends_on = [aws_ami_launch_permission.share_to_uat, aws_kms_key_policy.cmk_uat_access]

  tags = {
    ManagedBy   = "terraform"
    Environment = "uat"
    Source      = "i-0cb8bae3be071dc0e"
  }
}

### Bước 7: Security Group ###
resource "aws_security_group" "webapp_uat" {
  provider    = aws.uat
  name        = "jp-uat-sg-webapp"
  description = "Webapp UAT - inbound/outbound managed manually"
  vpc_id      = "vpc-05f30c1c1430f39a6"

  tags = { Name = "jp-uat-sg-webapp"; Environment = "uat" }
}

resource "aws_vpc_security_group_egress_rule" "deny_all" {

Mục tiêu & Trigger

Runbook này giải quyết vấn đề gì và khi nào phải dùng.

Vấn đề cốt lõi

CloudWatch Agent trên Windows EC2 dùng PID của IIS Worker Process làm dimension name. Mỗi lần IIS app pool recycle → PID mới → metric dimension mới được tạo → chi phí tăng vô hạn theo thời gian.

→Dấu hiệu nhận biết

Dấu hiệuCách kiểm traNgưỡng cảnh báo
Custom metric dimensions tăng liên tục CloudWatch → Metrics → namespace JP/CRM/Webapp → đếm số dimensions > 1,000 dims → cần xem xét
Chi phí CloudWatch bất thường AWS Cost Explorer → filter CloudWatch → xem MetricMonitorUsage > $50/tháng cho non-prod metrics
W3SVC_W3WP có nhiều instance names dạng số CloudWatch Metrics → W3SVC_W3WP → xem instance dimension values Instance names là số nguyên (PID) → cần fix ngay
Trong phạm vi

Sửa CWAgent config trên Windows EC2 (IIS) để loại bỏ PID-based dimensions. Áp dụng cho jp:prod webapp.

Ngoài phạm vi

Xóa metrics cũ đã tích lũy (tự expire sau 2 tuần). Thay đổi application code. Sửa IIS configuration.

Các bước thực hiện

Thực hiện đúng thứ tự. Dừng ngay nếu kết quả không như mong đợi.

🔴
ĐÂY LÀ PRODUCTION (jp:prod — 347486023795) — xác nhận lần 2 trước bước 2 và 3
  1. Minify config — bỏ _comment fields

    SSM Standard tier giới hạn 4,096 ký tự. File v2 gốc có _comment nên vượt quá. Cần strip trước khi upload.

    🟢 local · pythonlocal
    python3 -c "
import json

with open('cw-agent-config-jp-prod-webapp-v2.json') as f:
    data = json.load(f)

def strip(obj):
    if isinstance(obj, dict):
        return {k: strip(v) for k, v in obj.items() if not k.startswith('_')}
    elif isinstance(obj, list):
        return [strip(i) for i in obj]
    return obj

result = json.dumps(strip(data), separators=(',', ':'))
print(f'Size: {len(result)} chars')
with open('cw-agent-config-jp-prod-webapp-v2-min.json', 'w') as f:
    f.write(result)
"
    Size: 2610 chars — dưới 4,096, sẵn sàng upload
  2. Upload config v2 lên SSM Parameter Store

    Overwrite SSM Parameter với config mới. CWAgent chưa nhận ngay — cần bước 3 để push.

    🔴 write · prodjp:prod
    aws ssm put-parameter \
  --name "AmazonCloudWatch-jp-prod-webapp" \
  --type String --overwrite \
  --value file://cw-agent-config-jp-prod-webapp-v2-min.json \
  --profile jp:prod --region ap-southeast-1
    {"Version": 5, "Tier": "Standard"} — Version tăng thêm 1
  3. Push config + restart CWAgent qua SSM Run Command

    Dùng --instance-ids trực tiếp, KHÔNG dùng --targets Key=tag:Name (có thể trả về TargetCount: 0).

    🔴 write · prodjp:prod
    aws ssm send-command \
  --document-name "AmazonCloudWatch-ManageAgent" \
  --parameters '{"action":["configure"],"mode":["ec2"],
    "optionalConfigurationSource":["ssm"],
    "optionalConfigurationLocation":["AmazonCloudWatch-jp-prod-webapp"],
    "optionalRestart":["yes"]}' \
  --instance-ids "i-0cb8bae3be071dc0e" \
  --profile jp:prod --region ap-southeast-1 \
  --query 'Command.{CommandId:CommandId,Status:Status,TargetCount:TargetCount}'
    {"CommandId": "...", "Status": "Pending", "TargetCount": 1}
  4. Chờ 20 giây rồi kiểm tra kết quả

  --command-id "<CommandId từ bước 3>" \

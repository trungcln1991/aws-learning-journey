Tại sao AWS? Tại sao bây giờ? Bạn đang ở đâu trong bản đồ này?

▼
Quan điểm từ 20 năm kinh nghiệm

§Dự án thực hành cuối tháng 5

Dự án thực tế

Tạo Lambda + EventBridge + SNS → Teams notification khi có EC2 instance nào đó stop/start trong jp:prod. Dùng Python 10 dòng. Đây là feature Japfa thực sự cần — làm xong là có giá trị ngay.

Python cơ bản cho Lambda · Học song song 
# Lambda function gửi Teams notification khi EC2 state change
import json, urllib.request

def lambda_handler(event, context):
    instance_id = event['detail']['instance-id']
    state = event['detail']['state']
    msg = f"⚠️ EC2 {instance_id} → {state}"
    
    # Gửi Teams webhook (10 dòng!)
    payload = json.dumps({"text": msg}).encode()
    req = urllib.request.Request(WEBHOOK_URL, payload)
    urllib.request.urlopen(req)

§Vocab cơ bản — AWS Core Terms

instance
= máy chủ ảo (server)
"Launch an EC2 instance" = Khởi động 1 máy chủ EC2
provision
= cấp phát / khởi tạo tài nguyên
"Provision a database" = Tạo mới 1 database
deploy
= triển khai (đưa code/app lên server)
"Deploy the application" = Triển khai ứng dụng
throughput
= lưu lượng xử lý được (data/giây)
"High throughput storage" = Lưu trữ xử lý nhiều data
latency
= độ trễ (thời gian response)
"Low latency" = Độ trễ thấp, phản hồi nhanh
resilient
= có khả năng chịu lỗi, phục hồi nhanh
"Resilient architecture" = Kiến trúc chịu lỗi tốt
scalable
= có thể mở rộng quy mô
"Scalable solution" = Giải pháp mở rộng được
stateless / stateful
= không lưu trạng thái / có lưu trạng thái
Security Group = stateful · NACL = stateless
propagate
= lan truyền (route propagation = tự học route)
"Route propagation enabled" = bật học route tự động
deprecate
= không còn được hỗ trợ nữa (cũ/bỏ)
"This feature is deprecated" = Tính năng này sẽ bị bỏ
ephemeral
= tạm thời, sẽ mất khi stop/terminate
"Ephemeral storage" = Lưu trữ tạm, mất khi tắt máy
idempotent
= chạy nhiều lần vẫn ra kết quả như nhau
Terraform apply nhiều lần = idempotent

Site-to-Site VPN

Tổng quan

2 Site-to-Site VPN với BGP dynamic routing và tự động failover LA → HC.

▾
Định nghĩa

AWS Site-to-Site VPN là kết nối IPSec mã hóa giữa on-premises và AWS. Japfa dùng 2 VPN connections — Long An (primary) và Hương Canh (backup) — cả hai terminate tại Transit Gateway và dùng BGP để trao đổi route tự động.

✓

VPN Auto-Heal Lambda: Japfa đã có Lambda tự giám sát tunnel status qua CloudWatch và tự reset (initiate IKE rekey) khi tunnel DOWN. RTO thực tế được rút từ ~10 phút xuống ~2 phút.

→Thông số kỹ thuật mỗi VPN tunnel

Protocol

IPSec IKEv1 / IKEv2 — AWS hỗ trợ cả hai; FortiGate Japfa dùng IKEv1

Encryption

AES-256 + SHA-256 — Phase 1 và Phase 2

Routing

BGP dynamic (không dùng static) — FortiGate advertise on-prem CIDRs, AWS advertise VPC CIDRs

Tunnel / Connection

2 tunnel per VPN connection (AWS tạo tự động để redundancy)

DPD (Dead Peer Detection)

Enabled — phát hiện tunnel DOWN và trigger IKE re-negotiate

Bandwidth

Max 1.25 Gbps per tunnel — đủ cho workload Japfa (hiện tại <50 Mbps)

Cấu hình qua Console

Xem VPN status, tunnel health và BGP neighbors qua Console.

▾
  1. Đăng nhập Console jp:network — xem VPN status

    SSO → jp:network (787309524943) → VPC → Site-to-Site VPN Connections.

    Thấy 2 connections: Long An và Hương Canh.

    FieldGiá trị bình thườngCần lo khi nào
    Stateavailabledeleted / pending = lỗi
    Tunnel 1 StatusUPDOWN → auto-heal sẽ xử lý; nếu >5ph → check FortiGate
    Tunnel 2 StatusUP (hoặc DOWN nếu không dùng)Cả 2 DOWN = mất kết nối site đó
  2. Xem chi tiết tunnel — Outside IP và Inside IP

    Click vào 1 VPN connection → tab Tunnel details.

    Mỗi tunnel có Outside IP (AWS public IP) và Inside CIDR (BGP peering address /30).

    Outside IP thay đổi khi delete/recreate VPN — cần update FortiGate config nếu đổi
  3. Xem BGP routes đã propagate vào TGW

    VPC → Transit Gateway Route Tables → tab Routes.

    Các route có Origin = Propagated là từ BGP VPN. So sánh LA và HC routes — HC sẽ có AS_PATH dài hơn (65000 × 5).

    Nếu không thấy propagated routes từ LA nhưng thấy từ HC → LA BGP session bị down
  4. Kiểm tra FortiGate BGP status (nếu cần debug sâu)

    SSH vào FortiGate Long An (10.82.254.254) và chạy:

    • get router info bgp summary — Xem BGP neighbors và state (Established/Active)
    • get vpn ike status — Xem IKE Phase 1 status
    • get vpn ipsec tunnel summary — Xem Phase 2 và traffic counters
    BGP state "Active" (không phải "Established") = tunnel UP nhưng BGP chưa negotiate xong

Cấu hình qua CLI

AWS CLI để xem và quản lý VPN connections từ jp:network.

▾

→Lệnh đọc — kiểm tra VPN status (🟢 An toàn)

AWS CLI · Describe VPN🟢 READ ONLY
# Xem tất cả VPN connections và tunnel status
aws ec2 describe-vpn-connections \
  --profile jp:network --region ap-southeast-1 \
  --query 'VpnConnections[*].{ID:VpnConnectionId,State:State,Type:Type,CGW:CustomerGatewayId,TGW:TransitGatewayId}' \
  --output table

# Xem chi tiết tunnel state của Long An
aws ec2 describe-vpn-connections \
  --vpn-connection-ids vpn-09980936f3e3afeed \
  --profile jp:network --region ap-southeast-1 \
  --query 'VpnConnections[0].VgwTelemetry[*].{OutsideIP:OutsideIpAddress,Status:Status,AcceptedRoutes:AcceptedRouteCount,LastChanged:LastStatusChange}' \
  --output table

# Xem chi tiết tunnel state của Hương Canh
aws ec2 describe-vpn-connections \
  --vpn-connection-ids vpn-03cf2ba0a1c0934db \
  --profile jp:network --region ap-southeast-1 \
  --query 'VpnConnections[0].VgwTelemetry[*].{OutsideIP:OutsideIpAddress,Status:Status,AcceptedRoutes:AcceptedRouteCount}' \
  --output table

# Xem Customer Gateways
aws ec2 describe-customer-gateways \
  --customer-gateway-ids cgw-04872d4024c134695 cgw-06e3ae7e2c7d27cfc \
  --profile jp:network --region ap-southeast-1 \
  --query 'CustomerGateways[*].{ID:CustomerGatewayId,IP:IpAddress,ASN:BgpAsn,State:State}' \
  --output table

# Check CloudWatch tunnel metric (xem UP/DOWN log)
aws cloudwatch get-metric-statistics \
  --namespace AWS/VPN \
  --metric-name TunnelState \
  --dimensions Name=VpnId,Value=vpn-09980936f3e3afeed \
  --start-time $(date -u -v-1H +%Y-%m-%dT%H:%M:%SZ) \
  --end-time $(date -u +%Y-%m-%dT%H:%M:%SZ) \
  --period 300 --statistics Maximum \
  --profile jp:network --region ap-southeast-1

→Reset tunnel IKE (Manual VPN auto-heal — 🟡 Non-prod / Maintenance)

⚠

Reset VPN tunnel gây gián đoạn ~30-60 giây cho tunnel đó. Nên làm khi tunnel DOWN và auto-heal chưa xử lý, hoặc cần force renegotiate IKE.

AWS CLI · Reset VPN Tunnel (manual IKE initiation)🟡 MAINTENANCE
# Lấy Outside IP của tunnel cần reset
aws ec2 describe-vpn-connections \
  --vpn-connection-ids vpn-09980936f3e3afeed \
  --profile jp:network --region ap-southeast-1 \
  --query 'VpnConnections[0].VgwTelemetry[*].OutsideIpAddress'

# Reset IKE Phase 1 của 1 tunnel (VPN sẽ re-negotiate)
aws ec2 reset-vpn-connection \
  --vpn-connection-id vpn-09980936f3e3afeed \
  --vpn-tunnel-outside-ip-address {{outside-ip-tunnel-1}} \
  --profile jp:network --region ap-southeast-1

# Theo dõi tunnel UP sau ~60s
aws ec2 describe-vpn-connections \
  --vpn-connection-ids vpn-09980936f3e3afeed \
  --profile jp:network --region ap-southeast-1 \
  --query 'VpnConnections[0].VgwTelemetry[*].{IP:OutsideIpAddress,Status:Status}'

Cấu hình qua Terraform

VPN connections và Customer Gateways trong module terraform/network/.

▾
Terraform · vpn.tfHCL
# Customer Gateway — Long An
resource "aws_customer_gateway" "long_an" {
  bgp_asn    = 65000             # FortiGate LA BGP ASN
  ip_address = "27.65.195.246"   # Public IP FortiGate LA — đổi nếu ISP đổi IP
  type       = "ipsec.1"
  tags = { Name = "jp-network-cgw-aps1-long-an", Site = "long-an" }
}

# Customer Gateway — Hương Canh
resource "aws_customer_gateway" "huong_canh" {
  bgp_asn    = 65000              # Cùng ASN — phân biệt bằng IP và AS-prepend
  ip_address = "116.104.87.186"   # Public IP FortiGate HC
  type       = "ipsec.1"
  tags = { Name = "jp-network-cgw-aps1-huong-canh", Site = "huong-canh" }
}

# VPN Connection — Long An (terminate tại TGW)
resource "aws_vpn_connection" "long_an" {
  customer_gateway_id   = aws_customer_gateway.long_an.id
  transit_gateway_id    = aws_ec2_transit_gateway.main.id  # TGW (không dùng VGW)
  type                  = "ipsec.1"
  static_routes_only    = false   # false = dùng BGP dynamic routing
  enable_acceleration   = false   # Accelerated VPN (Global Accelerator) — chưa bật
  tags = { Name = "jp-network-vpn-aps1-long-an" }
}

# VPN Connection — Hương Canh
resource "aws_vpn_connection" "huong_canh" {
  customer_gateway_id = aws_customer_gateway.huong_canh.id
  transit_gateway_id  = aws_ec2_transit_gateway.main.id
  type                = "ipsec.1"
  static_routes_only  = false
  tags = { Name = "jp-network-vpn-aps1-huong-canh" }
}
# AS-prepend 5× cấu hình phía FortiGate HC, không phải AWS side

Ưu & Nhược điểm

Site-to-Site VPN vs Direct Connect cho kết nối on-prem Japfa.

▾

Ưu điểm

  • Chi phí thấp — ~$36/tháng per connection vs Direct Connect $100-200+
  • Redundancy 2 site — LA + HC với BGP failover tự động <5 phút
  • BGP dynamic routing — Không cần update route thủ công khi thêm subnet
  • Mã hóa IPSec end-to-end — AES-256 toàn bộ traffic qua internet
  • VPN Auto-Heal — Lambda tự phục hồi tunnel DOWN
  • 2 tunnel per connection — AWS cấp 2 tunnel mỗi connection để redundancy

Nhược điểm / Rủi ro

  • Phụ thuộc Internet ISP — VPN đi qua internet, không như Direct Connect dedicated
  • Latency cao hơn Direct Connect — Internet path không guaranteed latency
  • Bandwidth capped 1.25 Gbps — Nếu cần >1 Gbps stable → cần Direct Connect
  • PSK management thủ công — Phải update FortiGate config nếu VPN recreated

→Khi nào nên xem xét Direct Connect

ℹ

Chi phí VPN Japfa rất hợp lý (~$74/tháng cho 2 connections redundant). Không có cách tối ưu thêm nhiều — VPN connection fee là cố định theo giờ.

→Cân nhắc Accelerated VPN

Accelerated VPN dùng AWS Global Accelerator để route qua mạng backbone AWS thay vì internet — giảm latency và packet loss. Chi phí thêm ~$18/tháng per connection. Chưa bật cho Japfa — xem xét nếu latency LA→HC failover chậm hơn expected.

Checklist Thêm / Sửa / Xóa

Checklist VPN operations — xác nhận 2 lần trước mọi thay đổi.

▾

→Checklist: Thêm VPN connection mới

  • Tạo Customer Gateway mới — Cần biết: public IP on-prem, ASN FortiGate, type = ipsec.1
  • Tạo VPN connection → TGW — static_routes_only = false (BGP), attach vào TGW Japfa
  • Download VPN config cho FortiGate — Format: "Fortinet FortiGate" → lấy PSK và Outside IPs
  • Cấu hình FortiGate — IPSec Phase 1/2, BGP neighbor với Inside IP của tunnel
  • Verify BGP established — FortiGate: get router info bgp summary → state = Established
  • Verify routes propagate vào TGW — AWS: search-transit-gateway-routes → thấy on-prem CIDR
  • Test connectivity — Ping từ on-prem mới tới EC2 private IP
  • Update VPN auto-heal Lambda — Thêm VPN ID mới vào monitoring config nếu cần auto-heal

→Checklist: Sửa Customer Gateway IP (ISP đổi IP)

  • Không sửa được CGW IP trực tiếp — AWS không cho phép modify CGW IP sau khi tạo
  • Tạo CGW mới với IP mới — Tạo Customer Gateway resource mới với IP mới
  • Tạo VPN connection mới tới TGW — Từ CGW mới → download config → cấu hình FortiGate
  • Verify VPN mới UP — Cả 2 tunnel UP, BGP Established
  • Xóa VPN + CGW cũ — Sau khi VPN mới OK hoàn toàn

→Checklist: Xóa VPN connection (cực kỳ nguy hiểm)

  • Verify site backup đang UP — Nếu xóa LA, HC phải đang UP và healthy
  • Thông báo trước — User on-prem và SAP Indonesia sẽ bị ảnh hưởng trong lúc failover
  • Xác nhận 2 lần với 3T — VPN là P1, xóa nhầm = mất kết nối
  • Sau khi xóa: verify routes failover sang backup site — Check TGW route table còn routes từ HC

Best Practices & Tài liệu

Nguyên tắc vận hành VPN Japfa và tài liệu tham khảo.

▾

→Best Practices

Reliability

  • Luôn có 2 sites VPN — LA + HC, không bao giờ chỉ có 1 site
  • BGP thay vì static route — Tự động propagate route khi thêm subnet on-prem mới
  • DPD enabled — Phát hiện tunnel DOWN nhanh hơn (không phải chờ keepalive timeout)
  • Monitor cả 2 tunnel per connection — AWS tạo 2 tunnel mỗi connection; track cả 2

Security

  • KHÔNG ghi PSK vào skill/git — PSK là credential bảo mật; lấy từ Console VPN config download
  • Rotate PSK định kỳ — Đặc biệt nếu có nhân viên network nghỉ việc
  • Monitoring tunnel state — CloudWatch alarm khi TunnelState = 0 (DOWN)

Operations

  • VPN Auto-Heal Lambda — Đã có, không cần monitor thủ công. Lambda tự reset tunnel DOWN
  • FortiGate config backup — Trước mọi thay đổi FortiGate: execute backup config ftp/sftp

→Gotchas & Pitfalls

PITFALL

CGW IP không thể sửa — Nếu ISP đổi IP → phải tạo CGW mới + VPN mới, không edit được. Tên convention: jp-network-cgw-aps1-long-an-v2 khi tạo lại.

PITFALL

AS-prepend cấu hình FortiGate HC, không phải AWS — AWS không có field "AS prepend" trên VPN connection. FortiGate HC route-map set as-path prepend 65000 × 5 trước khi advertise sang AWS.

GOTCHA

Tunnel 2 thường DOWN — AWS cấp 2 tunnel nhưng BGP chỉ Establish 1 tại 1 thời điểm (không dùng ECMP cho same VPN connection). Tunnel 2 là hot standby — DOWN là bình thường.

TIP

Download FortiGate config ngay sau khi tạo VPN — File config từ Console có PSK, outside IPs, inside IPs, BGP config template sẵn. Download một lần, lưu nơi an toàn. KHÔNG để trên GitHub.

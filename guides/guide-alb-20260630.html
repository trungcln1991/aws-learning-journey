Định nghĩa

AWS ALB (Application Load Balancer) là Layer 7 load balancer — route HTTP/HTTPS traffic dựa theo host/path/header rules tới Target Group (EC2 instances). Japfa dùng 2 ALB: Public cho internet users và Internal cho API Gateway → EC2.

Cấu hình qua Console

Xem ALB status, target health và listener rules qua Console.

▾
  1. Đăng nhập Console jp:prod — xem ALB

    SSO → jp:prod (347486023795) → EC2 → Load Balancers.

    Thấy 2 ALB: Public và Internal. Click vào từng cái để xem chi tiết.

    FieldGiá trị bình thườngCần lo khi nào
    Stateactiveprovisioning / failed
    DNS Name*.ap-southeast-1.elb.amazonaws.comThay đổi nếu ALB recreated
    Availability Zonesap-southeast-1a + ap-southeast-1bChỉ 1 AZ = SPOF
  2. Kiểm tra Target Group Health

    EC2 → Target Groups → chọn TG → tab Targets.

    Mỗi target (EC2 instance) có health status: healthy / unhealthy / draining.

    Target "unhealthy" = Health check fail. Xem "Health check details" để biết lý do: timeout, HTTP 5xx, connection refused

    Health check config quan trọng:

    • Protocol: HTTP (IIS không cần HTTPS cho internal health check)
    • Path: / hoặc path cụ thể của từng app
    • Threshold: Healthy = 2 lần OK, Unhealthy = 3 lần fail
    • Timeout: 5-10 giây per check
  3. Xem Listener Rules

    ALB → tab Listeners and rules → click vào listener (:443 HTTPS).

    Xem danh sách rules: priority, conditions (host/path), actions (forward to TG, redirect, return fixed response).

    Rule priority: số nhỏ hơn = ưu tiên cao hơn. Rule "default" (priority last) = xử lý nếu không match rule nào
  4. Xem Access Logs (nếu incident)

    ALB → Attributes → Access logs → S3 bucket location.

    Log format: timestamp, client IP, request URL, response code, target IP:port, processing time.

    ALB access log có thể tắt để tiết kiệm S3 cost. Bật lại khi cần debug traffic pattern

Cấu hình qua CLI

AWS CLI để kiểm tra ALB và target health từ jp:prod.

▾
🔴

⚠️ ĐÂY LÀ PRODUCTION: jp:prod. Tất cả lệnh đọc OK. Lệnh sửa (modify-listener, register-targets, deregister-targets) phải xác nhận 2 lần.

→Lệnh đọc (🟢 An toàn)

AWS CLI · Describe ALB & Target Health🟢 READ ONLY · jp:prod
# Xem tất cả ALBs
aws elbv2 describe-load-balancers \
  --profile jp:prod --region ap-southeast-1 \
  --query 'LoadBalancers[*].{Name:LoadBalancerName,Scheme:Scheme,State:State.Code,DNS:DNSName}' \
  --output table

# Xem Target Groups
aws elbv2 describe-target-groups \
  --profile jp:prod --region ap-southeast-1 \
  --query 'TargetGroups[*].{Name:TargetGroupName,Port:Port,Protocol:Protocol,ALB:LoadBalancerArns[0]}' \
  --output table

# Kiểm tra health của targets trong 1 Target Group (thay {{tg-arn}} bằng TG ARN)
aws elbv2 describe-target-health \
  --target-group-arn {{tg-arn}} \
  --profile jp:prod --region ap-southeast-1 \
  --query 'TargetHealthDescriptions[*].{ID:Target.Id,Port:Target.Port,Health:TargetHealth.State,Reason:TargetHealth.Reason}' \
  --output table

# Xem Listener Rules của Public ALB
aws elbv2 describe-rules \
  --listener-arn {{listener-arn-https-443}} \
  --profile jp:prod --region ap-southeast-1 \
  --query 'Rules[*].{Priority:Priority,Conditions:Conditions,Actions:Actions}' \
  --output json

# Xem ALB attributes (idle timeout, access logs, ...)
aws elbv2 describe-load-balancer-attributes \
  --load-balancer-arn arn:aws:elasticloadbalancing:ap-southeast-1:347486023795:loadbalancer/app/jp-prod-alb-pb-aps1-public/185b2f6d35081736 \
  --profile jp:prod --region ap-southeast-1 \
  --output table

→Deregister / Register target (🔴 Production)

AWS CLI · Register/Deregister Target🔴 WRITE · jp:prod
# Deregister target (trước khi patch EC2 webapp)
# ⚠️ TẤT CẢ TRAFFIC SẼ DỪNG khi deregister hết target
aws elbv2 deregister-targets \
  --target-group-arn {{tg-arn}} \
  --targets Id=i-0cb8bae3be071dc0e \
  --profile jp:prod --region ap-southeast-1

# Register lại sau khi patch xong
aws elbv2 register-targets \
  --target-group-arn {{tg-arn}} \
  --targets Id=i-0cb8bae3be071dc0e,Port=62375 \
  --profile jp:prod --region ap-southeast-1

# Chờ target healthy sau register
aws elbv2 describe-target-health \
  --target-group-arn {{tg-arn}} \
  --profile jp:prod --region ap-southeast-1 \
  --query 'TargetHealthDescriptions[*].{ID:Target.Id,Health:TargetHealth.State}'

Cấu hình qua Terraform

ALB, Target Groups và Listener Rules trong TF jp:prod.

▾
Terraform · alb.tfHCL
# Public ALB (internet-facing + WAF)
resource "aws_lb" "public" {
  name               = "jp-prod-alb-pb-aps1-public"
  internal           = false              # internet-facing
  load_balancer_type = "application"      # ALB (Layer 7)
  security_groups    = [aws_security_group.alb_public.id]
  subnets            = [subnet_public_1a_id, subnet_public_1b_id]  # 2 AZ required
  drop_invalid_header_fields = true        # Security: drop malformed headers

  access_logs {
    bucket  = "jp-prod-alb-logs-aps1"    # S3 bucket cho access logs
    enabled = true
  }
  tags = { Name = "jp-prod-alb-pb-aps1-public", WAFAttached = "true" }
}

# Internal ALB (API Gateway VPC Link)
resource "aws_lb" "internal" {
  name               = "jp-prod-alb-pv-aps1-internal"
  internal           = true               # private — chỉ truy cập trong VPC
  load_balancer_type = "application"
  security_groups    = [aws_security_group.alb_internal.id]
  subnets            = [subnet_private_app_1a_id, subnet_private_app_1b_id]
  tags = { Name = "jp-prod-alb-pv-aps1-internal" }
}

# Target Group — BankSystem port 62375
resource "aws_lb_target_group" "banksystem" {
  name                 = "jp-prod-tg-banksystem"
  port                 = 62375
  protocol             = "HTTP"          # EC2 IIS → HTTP internally (HTTPS ở ALB level)
  vpc_id               = data.aws_vpc.prod.id
  deregistration_delay = 60             # Draining timeout trước khi remove target

  health_check {
    enabled             = true
    path                = "/"
    port                = "traffic-port"
    healthy_threshold   = 2
    unhealthy_threshold = 3
    timeout             = 5
    interval            = 30
    matcher             = "200,302"       # IIS đôi khi trả 302 redirect
  }
}

# Register EC2 webapp vào Target Group
resource "aws_lb_target_group_attachment" "banksystem_webapp" {
  target_group_arn = aws_lb_target_group.banksystem.arn
  target_id        = "i-0cb8bae3be071dc0e"  # EC2 webapp ID
  port             = 62375
}

# Listener HTTPS :443 trên Public ALB
resource "aws_lb_listener" "https" {
  load_balancer_arn = aws_lb.public.arn
  port              = 443
  protocol          = "HTTPS"
  ssl_policy        = "ELBSecurityPolicy-TLS13-1-2-2021-06"  # TLS 1.2/1.3 only
  certificate_arn   = aws_acm_certificate.main.arn

  default_action {
    type             = "forward"
    target_group_arn = aws_lb_target_group.banksystem.arn
  }
}

Ưu & Nhược điểm

ALB so với NLB và API Gateway trực tiếp.

▾

Ưu điểm

  • Layer 7 routing — Route theo path/host/header — 1 ALB phục vụ nhiều app
  • TLS termination — HTTPS tại ALB; EC2 chỉ cần HTTP — đơn giản cert management
  • WAF integration — Attach WAF trực tiếp vào Public ALB
  • Multi-AZ HA — SLA 99.9%, auto cross-AZ routing
  • Health check built-in — Auto remove unhealthy target mà không cần monitor thêm
  • Access logs — Log toàn bộ request với client IP, latency, response code

Nhược điểm / Rủi ro

  • Single EC2 target SPOF — Chỉ 1 EC2 webapp; nếu EC2 unhealthy thì tất cả app down
  • Không có auto-scaling — Không có ASG phía sau ALB — spike traffic = EC2 quá tải
  • Chi phí data — LCU pricing tăng theo số connection, bandwidth, rules
  • SSL cert renewal — ACM cert cần renew; nếu expire → HTTPS break

Checklist Thêm / Sửa / Xóa

Checklist ALB operations trên Production — xác nhận 2 lần.

▾

→Checklist: Thêm Target Group / App mới

  • Xác định port IIS mới — Verify IIS đang listen đúng port trên EC2 webapp
  • Tạo Target Group — Protocol HTTP, port IIS, health check path, matcher 200 hoặc 200,302
  • Register EC2 webapp vào TG — aws elbv2 register-targets
  • Verify target healthy — Chờ health check pass (2 lần healthy) trước khi thêm rule
  • Thêm Listener Rule — Priority thấp hơn rule hiện có, condition path/host, action forward TG
  • Cập nhật SG inbound — SG EC2 webapp phải allow port IIS mới từ SG ALB (không phải 0.0.0.0/0)

→Checklist: Patch EC2 (deregister trước)

  • Thông báo maintenance window — BankSystem/CRM/E-Approval/FeedProd sẽ tạm ngừng
  • Deregister target khỏi tất cả TG — Đợi draining timeout (60s) để requests hiện tại hoàn thành
  • Patch / restart IIS / update app — Thực hiện thay đổi trên EC2
  • Register lại EC2 vào tất cả TG — Với đúng port cho từng TG
  • Verify tất cả TG healthy — Tất cả targets phải healthy trước khi xác nhận hoàn thành

Best Practices & Tài liệu

Nguyên tắc vận hành ALB Japfa.

▾

→Best Practices

Security

  • TLS 1.2+ only — Policy ELBSecurityPolicy-TLS13-1-2-2021-06 — không cho TLS 1.0/1.1
  • drop_invalid_header_fields = true — Block request có header bất thường
  • WAF chỉ attach Public ALB — Internal ALB không cần WAF (chỉ accessible trong VPC)
  • SG ALB → SG EC2 (không dùng CIDR) — Rule inbound EC2 webapp: allow from SG-ALB, không phải 0.0.0.0/0

Reliability

  • Health check timeout hợp lý — IIS khởi động chậm: set healthy_threshold = 2, unhealthy = 3
  • Deregistration delay 60s — Đủ thời gian request đang xử lý hoàn thành trước khi remove target
  • Monitor HealthyHostCount >= 1 — Alarm khi 0 healthy hosts

Cost

  • 1 ALB nhiều TG — Không tạo riêng ALB cho từng app; dùng path/host routing
  • Access logs tắt khi không cần — S3 ingest cost tuy nhỏ nhưng có thể tắt khi stable

→Gotchas

PITFALL

IIS health check trả 302 — IIS redirect / đến /?aspxerrorpath=/ trả 302 chứ không phải 200. Health check matcher phải là "200,302" không thì target sẽ unhealthy liên tục.

PITFALL

WAF chỉ gắn Public ALB không phải API Gateway — WAF jp-prod-waf-aps1-webapp attach vào ALB ARN, không phải API Gateway resource. Kiểm tra kỹ ARN khi sửa WAF association.

GOTCHA

SSL cert expire — ACM cert tự động renew nếu DNS validation còn valid. Nếu Route 53 record bị xóa → cert không renew → HTTPS expire. Monitor cert expiry qua Certificate Manager console.

TIP

ALB ARN cố định — arn:aws:elasticloadbalancing:ap-southeast-1:347486023795:loadbalancer/app/jp-prod-alb-pb-aps1-public/185b2f6d35081736 — dùng ARN này khi gắn WAF.
